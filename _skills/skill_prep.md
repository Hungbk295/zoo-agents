# Skill: prep

Use before starting any task. Fresh context every time (Ralph Wiggum Loop).

1. Read the task in `data.json` — understand id, title, priority, deadline.
2. Read the linked RD in `rds/` — understand requirements, done criteria, any revision notes.
3. Check if skills are listed in the RD → read those skill files.
4. If anything is unclear, ask for clarification before starting.
5. Update data.json status → `in_progress`.

**Rule:** Never carry over assumptions from a previous task. Read everything fresh.
